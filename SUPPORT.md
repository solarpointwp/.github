# Support

The following guidance explains how to get help with SolarPoint projects.

## Reporting Security Vulnerabilities

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.** If you believe you have discovered a security vulnerability in a SolarPoint project, please refer to our [Security Policy](https://github.com/solarpointwp/.github/security/policy) for instructions on how to responsibly disclose it.

## Getting Help

For general support questions, how-to guidance, or best practices, use the **Discussions** tab of the relevant repository.

> [!NOTE]
> The **Issues** tab is reserved strictly for reproducible bug reports and feature requests. Support questions opened as issues may be converted to discussions.

**Before asking:**

- Read and adhere to our [Code of Conduct](https://github.com/solarpointwp/.github?tab=coc-ov-file).
- Check the README and project documentation.
- Search the **Issues** and **Discussions** tabs of the repository first.

### Asking Quality Questions

A well-framed question makes it easier for others to provide a useful answer.

- **Define the problem clearly:** What is the goal, and what is happening instead?
- **Show what has already been tried:** Include steps taken to solve the problem.
- **Provide context:** Include relevant code snippets, error messages (as text, not screenshots), PHP version, WordPress version, and any other relevant environment details.

## Support Policy

SolarPoint is maintained by volunteers. While every effort is made to respond in a timely manner, responses are not guaranteed. Please be patient and respectful when waiting for a reply.

Issues and pull requests labeled `needs info` will be **marked as stale after 30 days of inactivity** and **closed 7 days later** if no response is received. Closed issues may be reopened if the problem persists.

## Commercial Support

Commercial support is not available at this time.
