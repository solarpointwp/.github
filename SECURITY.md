# Security

SolarPoint takes the security of its software seriously and will work to address any verified vulnerabilities promptly.

Thank you for helping make SolarPoint safe for everyone.

## Reporting Security Issues

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, please send an email to [security@solarpointwp.com](mailto:security@solarpointwp.com).

### Disclosure Policy

SolarPoint does not currently offer a bug bounty program, but credit will be given in release notes for responsible disclosures. Reports will be acknowledged within 7 days. Status updates will follow as the issue is investigated and resolved. If no response is received, please follow up via email to ensure the original message was delivered. Vulnerability details should be kept confidential until a fix has been released.

### What to Include in Your Report

Please include the following information to help triage the report more quickly:

- Type of issue (e.g. buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit the issue
